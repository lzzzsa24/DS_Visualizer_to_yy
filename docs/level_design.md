# 🗺️ 关卡设计指南

* **位置**: `resources/maps/*.txt`
* **格式**: UTF-8 纯文本

## 1. 符号对照表 (Legend)

| 符号 | ID | 元素 | 交互逻辑 |
| :--: | :--: | :--- | :--- |
| **`#`** | 1 | 🧱 墙壁 | 不可通行 |
| **`.`** | 0 | ⬜ 地板 | 可通行 |
| **` `** | -1 | 🌌 虚空 | (空格) 隐形边界，不渲染 |
| **`P`** | 0 | 😐 玩家 | 出生点 (Player Start) |
| **`D`** | 8 | 🚪 大门 | **通关点** (栈顶是钥匙) |
| **`K`** | 5 | 🔑 钥匙 | 道具 - 用于开门 |
| **`S`** | 4 | ⚔️ 宝剑 | 道具 - 用于杀怪 |
| **`W`** | 3 | 🧪 水 | 道具 - 用于灭火 (代码映射字符为 W) |
| **`M`** | 7 | 👾 怪物 | **需剑在栈顶** (剑消耗) |
| **`F`** | 6 | 🔥 火焰 | **需水在栈顶** (水消耗) |

## 2. 设计规则

1.  **栈机制 (Stack LIFO)**
    * 玩家只能使用背包**最上方**的道具。
    * *陷阱设计*：诱导玩家先捡错误的道具压住关键道具，迫使其重新规划路径。

2.  **不规则地图**
    * 使用**空格**填充地图轮廓外的区域。
    * 行长度无需对齐，程序会自动补全。
